Connector 部署说明 推荐直接从 GitHub Release 下载构建产物

构建产物清单

每次 main 分支构建成功后会创建一个 GitHub Release，tag 是 build-<SHA>。Connector 机器只需要下载 connector 镜像包。

GitHub Release: build-<SHA>
  testflying-connector-<SHA>.tar.gz          # Connector 机器只需要这个
  testflying-server-<SHA>.tar.gz             # 中心后台 API 镜像
  testflying_server-0.1.0-py3-none-any.whl   # 后端 wheel

Actions artifact 备用入口:
  testflying-build-<SHA>/
    images/testflying-connector-<SHA>.tar.gz

1. 从 GitHub Release 下载并加载 Connector 镜像

SHA=<commit sha>

mkdir -p /opt/testflying/artifacts
cd /opt/testflying/artifacts

gh release download "build-$SHA" \
  -R baoluchuling/testflying-api \
  -p "testflying-connector-$SHA.tar.gz" \
  -D .

gunzip -c "testflying-connector-$SHA.tar.gz" | docker load
docker tag "testflying-connector:$SHA" testflying-connector:local

备用：从 GitHub Actions artifact 下载

RUN_ID=<GitHub Actions run id>
SHA=<commit sha>

gh run download "$RUN_ID" \
  -R baoluchuling/testflying-api \
  -D .

ls -R "testflying-build-$SHA"
gunzip -c "testflying-build-$SHA/images/testflying-connector-$SHA.tar.gz" | docker load
docker tag "testflying-connector:$SHA" testflying-connector:local

2. 先用 mock 模式验证

docker rm -f testflying-connector || true

docker run -d --name testflying-connector \
  -p 8100:8100 \
  --memory=64m \
  -e TESTFLYING_CONNECTOR_DEVELOPER_ACCOUNT_ID={{ account.id }} \
  -e TESTFLYING_CONNECTOR_TOKEN=<random-token> \
  testflying-connector:local

curl http://127.0.0.1:8100/health

3. Apple 账号 live 模式

docker rm -f testflying-connector || true

docker run -d --name testflying-connector \
  -p 8100:8100 \
  --memory=64m \
  -v /opt/testflying/secrets/apple-a:/run/secrets/apple:ro \
  -e TESTFLYING_CONNECTOR_STORE_MODE=live \
  -e TESTFLYING_CONNECTOR_DEVELOPER_ACCOUNT_ID={{ account.id }} \
  -e TESTFLYING_CONNECTOR_TOKEN=<random-token> \
  -e TESTFLYING_CONNECTOR_APPLE_ISSUER_ID=<issuer-id> \
  -e TESTFLYING_CONNECTOR_APPLE_KEY_ID=<key-id> \
  -e TESTFLYING_CONNECTOR_APPLE_PRIVATE_KEY_PATH=/run/secrets/apple/AuthKey_XXXX.p8 \
  testflying-connector:local

4. Google 账号 live 模式

docker rm -f testflying-connector || true

docker run -d --name testflying-connector \
  -p 8100:8100 \
  --memory=64m \
  -v /opt/testflying/secrets/google-a:/run/secrets/google:ro \
  -e TESTFLYING_CONNECTOR_STORE_MODE=live \
  -e TESTFLYING_CONNECTOR_DEVELOPER_ACCOUNT_ID={{ account.id }} \
  -e TESTFLYING_CONNECTOR_TOKEN=<random-token> \
  -e TESTFLYING_CONNECTOR_GOOGLE_SERVICE_ACCOUNT_JSON_PATH=/run/secrets/google/service-account.json \
  testflying-connector:local
Connector 地址填这台机器对中心后台可访问的地址，例如 http://<connector-host>:8100。调用 Token 必须和 TESTFLYING_CONNECTOR_TOKEN 一致；账号 ID 必须保持为 {{ account.id }}，否则检查连接会返回 403。
